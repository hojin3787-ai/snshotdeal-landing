# 원페이저 — 예시 모음

> 본문(가이드)에서는 예시는 제외됩니다. 필요한 경우 아래 예시를 참고하세요.

## 1. 문제 정의 예시
```markdown
1. 모호한 정체성 - "무엇을 하는 앱인지 모르겠다"
2. 낮은 당첨 확률 - 당첨자 없는 회차 빈번
3. 느린 보상 - 포인트 모으기까지 평균 30일 소요
```

## 2. 시장 인사이트 예시
```markdown
- 수익목적형(설문, 게임): 피로도 높음
- 생활연동형(걷기, 충전): 진입 장벽 낮음
- 걷기형은 경쟁 포화 / 충전형은 경쟁자 부재
```

## 3. 타겟/페르소나 예시
```markdown
앱테크에 피로감을 느낀 20-30대 직장인/대학생
```

```markdown
**지친 앱테크러 민수 (28세, 직장인)**
> "캐시워크, 럭키즈 다 해봤는데... 걷기는 귀찮고,
> 로또는 당첨 안 되고... 간단하게 매일 확실히 받을 수 있는 앱 없을까요?"
```

## 4. 가치 제안/핵심 가설 예시
```markdown
- 방치형 시스템: 충전만 하면 자동으로 보상 획득
- 투명한 확률: 목표 달성 시 반드시 당첨자 1명 배출
- 즉시 보상: 기프티콘 즉시 구매
```

```markdown
"명확한 진입점(충전) + 다양한 선택지(응모/콘텐츠/기프티콘) = 높은 리텐션"
```

## 5. 차별점 예시
```markdown
### vs 캐시는내차지
- 경쟁사: 충전 → 응모만 가능
- 우리: 충전 → 응모 + 콘텐츠 + 기프티콘 (다양한 선택지)

### vs 럭키즈
- 경쟁사: 실제 로또 연동 → 당첨자 없는 회차 빈번
- 우리: 목표 달성 시 즉시 추첨 → 반드시 1명 당첨
```

## 6. 핵심 지표/목표 예시
```markdown
3일 내 콘텐츠를 10회 이상 즐긴 사용자 수
→ 단순 가입이 아닌 "리워드 경험"을 한 유저
```

```markdown
- D3 잔존율: 20% 이상
- 일일 평균 콘텐츠 참여 횟수: 5회 이상/DAU
- 가입 후 첫 참여까지 소요 시간: 평균 5분 이내
```

```markdown
2025년 12월 31일까지
- 총 다운로드: 10,000회
- 오가닉 유입 중심 + ASO 최적화
```

## 8. 가설 & 리스크 예시
```markdown
- G1: 쉬운 참여 > 높은 보상 - 보상이 적어도 참여 장벽 낮으면 리텐션 높은가?
- G2: 소확행 vs 로또 - 소액 확정 보상 vs 고액 확률 보상, 뭘 선호하나?
- G3: 다양성 효과 - 다양한 선택지가 단일 경로보다 전환율 높은가?
```

```markdown
- 기술: 배터리 감지 정확도 → 충분한 테스트, 에러 처리
- 수익: 광고 수익 저조 → eCPM 모니터링, 최적화
```
