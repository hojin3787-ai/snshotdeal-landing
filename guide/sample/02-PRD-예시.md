# PRD — 예시 모음

> 본문(가이드)에서는 예시는 제외되며, 필요한 경우 아래 예시를 참고하세요.

## 1. 개요 예시
```markdown
이 문서는 **캐시충전 앱의 제품 요구사항**을 정의합니다.
왜 만드는지, 누가 쓰는지, 무엇을 만드는지, 어떻게 측정할지를 명시합니다.
```

```markdown
"휴대폰 충전하면 캐시도 충전되는 방치형 앱테크"
```

```markdown
- 진입점: 휴대폰 충전
- 재화: 에너지 → 포인트
- 사용처: 경품 응모 + 기프티콘
- 차별점: 명확한 정체성
```

## 2. 배경 예시
```markdown
| 유형 | 특징 | 예시 | 문제점 |
|---|---|---|---|
| 수익목적형 | 추가 행동 필요 | 설문, 게임 | 사용자 피로도 높음 |
| 생활연동형 | 일상 행동에 보상 | 걷기, 충전 | 낮은 진입 장벽 |
```

## 3. 솔루션 예시
```markdown
**캐시충전 = 휴대폰 충전하면 캐시도 충전되는 방치형 앱테크**

휴대폰을 충전하면 에너지를 획득하고, 이를 활용해 경품 응모, 콘텐츠 참여...
```

```markdown
- 대부분의 사람이 하루에도 여러 번 수행
- 앱에서 감지 가능
- 경쟁자 부재
```

```markdown
1. 휴대폰 충전하면 내 지갑도 충전돼요
2. 조건없이 에너지 지급!
3. 에너지로 응모해서 상품권 받아가세요!
```

```markdown
- 핵심 가치: "충전 = 보상"
- 차별점: "조건없이"
- 행동 유도: "응모", "콘텐츠 즐기기"
```

## 4. 타겟 유저 예시
```markdown
**앱테크에 피로감을 느낀 20-30대**

- 연령대: 20-30대 직장인, 대학생
- 주요 불만: 당첨 확률 낮음, 복잡한 구조
```
