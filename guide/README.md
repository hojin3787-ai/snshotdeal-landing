# 📚 문서 작성 가이드

프로젝트 문서 작성을 위한 **사용자 가이드**입니다.

---

## 📝 문서 작성 순서

1. **원페이저** → 왜 만드나?
2. **PRD** → 무엇을 만드나?
3. **서비스정책** 🔄 → 어떻게 작동하나? (수치/규칙, Living Document)
4. **화면설계** → 어떤 흐름? 어떤 구조?
5. **디자인정책** → 어떤 모습?
6. **개발설계** → 어떻게 만드나?
7. **테스트계획** → 어떻게 검증하나?

> 🔄 서비스정책 = Living Document (각 단계 완료 시 계속 업데이트)

## 💡 빠른 시작 (대화형 · 순차 진행)

### 1. 아이디어 말하기 → AI가 질문
- 예: "배달 음식 리뷰 앱을 만들고 싶어"
- AI가 필요한 정보를 질문하고, 당신이 답합니다.

### 2. 섹션 단위로 작성·검토
- 한 번에 한 섹션만 작성합니다. (순차 진행)
- 섹션 초안 → 검토/수정 → 승인 후 다음 섹션으로 진행합니다.
- 승인 키워드: "[섹션 제목] 섹션 진행해" (예: "문제 정의 섹션 진행해")

### 3. 산출물 저장 위치
- `outputs/docs/` 폴더
- 구체적 수치/제한/규칙은 `03-서비스정책.md`에만 작성

### 4. 다음 단계 진행
- 각 단계는 이전 단계 문서 승인 후 진행합니다.
- 화면설계는 Gate A(Lo-fi) 승인 후 디자인으로 이동합니다.
- 5단계(디자인정책) 이후 단계는 4단계 승인(Gate A) 전에는 논의/작성하지 않습니다.
  - 화면설계 내 순서: 화면 목록(섹션 4) → 공통 레이아웃(섹션 5) → 화면정의(Index, 섹션 6) → (개별) 화면정의 작성 → Lo‑fi(섹션 7)

### 5. 예시 모음
- 모든 예시는 `guide/sample/` 폴더에서 확인하세요.

### 6. 추가 가이드
- 화면정의 가이드: `guide/04-화면설계-가이드.md` 섹션 6 참조

## ⚠️ 반드시 지켜야할 원칙

### 단일 진실 공급원 (Single Source of Truth)

**모든 구체적인 수치는 서비스정책 문서에만 작성합니다.**

| 문서 | 작성하는 것 | 작성하지 않는 것 |
|---|---|---|
| **서비스정책** | "30초마다 +1", "에너지 5개", "일일 10회" | 화면 구성, UI 상세 |
| **PRD** | "일정 시간마다", "에너지 지급" (개념) | 구체적 수치 |
| **화면설계** | 사용자 흐름, 화면 구조 | 구체적 수치 |
| **사용자 흐름** | 흐름 로직, 분기 조건 | 구체적 수치, 페이지 구성 상세 |
| **디자인정책** | 디자인 시스템, UI 스펙 | 비즈니스 정책 수치 |

**왜 이렇게 하나요?**
- ✅ 정책 변경 시 서비스정책 문서 한 곳만 수정
- ✅ 문서 간 수치 불일치 없음
- ✅ 역할 명확: 서비스정책=수치, 다른 문서=개념/흐름

### 순차 진행(Section Gating) 원칙
- 한 턴에 하나의 섹션만 작성합니다.
- "전체 작성" 요청이 와도 거절하고 다음 섹션을 물어봅니다.
- 다음 섹션으로 이동은 "진행해 섹션 N" 승인 후 수행합니다.

**예시**:
```markdown
✅ PRD: "사용자는 주기적으로 에너지를 획득합니다"
✅ 사용자 흐름: "에너지 지급 → 상태 업데이트"
✅ 서비스정책: "30초마다 에너지 +1, 일일 최대 300회"

❌ PRD: "30초마다 에너지 +1"
❌ 사용자 흐름: "에너지 5개 차감"
```

## ❓ FAQ

**Q: 구체적인 수치를 어디에 작성하나요?**
- A: 서비스정책 문서에만 작성합니다. 다른 문서는 개념적 표현만 사용하고 서비스정책을 참조하세요.

**Q: 사용자 흐름에 페이지 구성을 작성할 수 있나요?**
- A: 아니요. 사용자 흐름은 흐름 로직만 작성합니다. 페이지 구성은 화면설계 본문에 작성하세요.

**Q: 서비스정책은 언제 업데이트하나요?**
- A: 각 단계 완료 시 검토하고 필요 시 업데이트합니다 (Living Document).

**Q: 순서를 바꿀 수 있나요?**
- A: 1→2→3→4 순서는 필수입니다. 개발설계는 화면설계와 병행 가능하지만, 각 문서는 섹션 단위로 순차 진행합니다.

**Q: 섹션 승인은 어떻게 하나요?**
- A: 섹션 제목로 알려주세요. 예: 사용자 흐름 완료 후 "정보 구조 섹션 진행해".
