# 서비스정책 작성 가이드

## 🚦 시작하기 (대화형 · 순차 업데이트)
- PRD 승인 후, 쓸 수 있는 섹션부터 하나씩 작성/업데이트합니다.
- 한 번에 한 섹션만 작성합니다. 승인 키워드: 섹션 제목로 승인 (예: "에너지 섹션 진행해").
- API/에러/디자인 세부는 본 문서에 작성하지 않습니다(각 설계서 참조).
- 본문에서는 예시를 제외합니다. 예시는 `guide/sample/03-서비스정책-예시.md`에서 확인하세요.

## 📌 이 문서는 무엇인가요?

**서비스정책**은 서비스/콘텐츠 운영에 필요한 **구체적인 수치, 제한, 규칙**을 정리한 문서입니다.

**위치**: `outputs/docs/03-서비스정책.md`

**특징**: 🔄 **살아있는 문서** (Living Document)
- PRD 후 초안 작성
- 이후 계속 업데이트

---

## 🔄 작성 철학

### "지금 쓸 수 있는 것만 쓰고, 필요할 때 업데이트"

**PRD 직후**: 기본 규칙만
- 회원: 가입/탈퇴/재가입
- 재화: 에너지/포인트 획득/사용/소멸
- 콘텐츠: 참여 조건, 보상
- 핵심 수치: "30초당 +1", "최대 10,000개"

**프로젝트 진행 중**:
- 새로운 콘텐츠 추가 시 → 참여 규칙 추가
- 정책 변경 결정 시 → 해당 섹션 수정
- 구체적 수치 확정 시 → 수치 업데이트

**테스트 후**: 최종 확정
- 최종 수치 조정
- 새로운 제한사항

---

## ✍️ 작성 방법 (요약)

1) 초안 작성( PRD 승인 직후 )
- 회원/재화/콘텐츠/충전/출석/광고/응모/상점/초대 중 “지금 확정된 것만” 작성
- 수치·제한·규칙은 구체적으로, 단 추측 금지

2) 단계별 업데이트 트리거
- 화면설계 후: 신규 콘텐츠/화면 기반 정책 추가 여부 확인
- 디자인/개발설계 후: 보통 변경 없음(디자인·API는 각 설계서)
- 테스트 후: 수치 미세 조정/새 제한 추가

3) 포함/제외 범위
- 포함: 회원/재화/참여 규칙, 일일/보유 제한, 지급/소멸 기준 등
- 제외: API 제약/에러 코드(→ 개발설계), UI 규칙(→ 디자인/화면설계), 법적 문서

4) 형식
- 간결한 문단/표 위주, 중복 금지
- 변경 이력 표는 매 업데이트 시 추가

### 3. 변경 이력 업데이트

매번 업데이트할 때마다:
```markdown
| 날짜 | 작성자 | 변경 내용 |
|---|---|---|
| 2025-10-20 | PM | 초안 작성 (PRD 기반) |
| 2025-10-25 | PM | 화면설계 기반 정책 추가 |
| 2025-11-01 | Dev | API 제한 추가 |
```

## 📋 주요 정책 영역

### 1. 회원
- 가입/탈퇴/재가입
- 계정 정지/복구

### 2. 재화
- **에너지**: 획득/사용/소멸/제한
- **포인트**: 획득/사용/소멸/제한

### 3. 충전
- 감지 조건
- 획득 주기
- 제약 사항

### 4. 출석 체크
- 시간대/보상
- 연속 출석 보너스

### 5. 광고
- 시청 조건/보상
- 하루 제한

### 6. 콘텐츠
- VS 투표
- OX 퀴즈
- 캐시 러쉬

### 7. 응모
- 참여 조건
- 추첨 시간/방식

### 8. 상점
- 구매 조건
- 환불 규칙

### 9. 초대
- 초대 보상
- 제한

## 📝 작성 팁

### DO ✅
- **구체적인 수치 명시**: "30초당 +1", "최대 10,000개"
- **표 활용**: 비교, 조건 정리
- **변경 이력 업데이트**: 매번 작성
- **현재 아는 것만**: 추측하지 않기
- **지속적 업데이트**: 각 단계마다 검토

### DON'T ❌
- **모호한 표현**: "적절히", "가능한", "일정 시간"
- **추측성 내용**: 아직 결정되지 않은 것
- **버전 번호**: v0.1, v1.0 같은 표기 (단순히 날짜로)
- **이용약관/개인정보**: 범위 벗어남

## 🔍 단계별 검토 체크리스트

### PRD 완료 후
- [ ] 회원 정책 (가입/탈퇴/재가입)
- [ ] 재화 정책 (에너지/포인트 기본 규칙)
- [ ] 콘텐츠 정책 (참여 조건, 보상)
- [ ] 변경 이력 업데이트

### 화면설계 완료 후
- [ ] 새로운 콘텐츠 추가됐나? → 참여 규칙 추가
- [ ] 새로운 재화 추가됐나? → 획득/사용 규칙 추가
- [ ] 변경 이력 업데이트

### 디자인 완료 후
- [ ] 서비스정책에 추가할 내용 있나? (보통 없음)
- [ ] 변경 이력 업데이트 (변경 있었다면)

### 개발설계 완료 후
- [ ] 서비스정책에 추가할 내용 있나? (보통 없음)
- [ ] 변경 이력 업데이트 (변경 있었다면)

### 테스트 완료 후
- [ ] 최종 수치 조정 필요한가?
- [ ] 새로운 제한사항 추가됐나?
- [ ] 변경 이력 업데이트

## 📖 용어 정의

### 서비스정책 vs 설계 vs 계획

**정책 (Policy)**:
- 규칙, 기준, 제한을 정의
- 예: 서비스정책, 디자인정책

**설계 (Design)**:
- 구조, 흐름, 아키텍처를 정의
- 예: 화면설계, 개발설계

**계획 (Plan)**:
- 전략, 범위, 방법을 정의
- 예: 테스트계획
